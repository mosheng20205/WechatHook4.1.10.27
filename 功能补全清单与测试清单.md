### XML 任务细分

- ✅ 确认旧 `Offsets` 命名空间(`FORWARD_XML_CALL 0x1CF3D20` 等)对 4.1.10.27 **已失效**:落在 Qt UI 布局函数 `sub_181CF2FA0` 内部(偏移 `0xD80`),硬编码 `g_weixinBase+0x367849` 亦落在函数中段,直接调用会跳进无关 UI 代码并崩溃 `Weixin.exe`。
- ✅ 用 IDA 重定位现代发送路径:XML/appmsg 必须走**已验证的统一** `send_message`(`0x1677A30`)+ 内容对象(vtable 决定消息类型)。内容对象工厂 `sub_182A1AC80` 按 `"<Type>MessageSendSource"` 类型名字符串分派,已解出映射并写入 `include/global.h`:
  - `len 17 "MessageSendSource"` → `0x84F9A78`(通用基类,`0x718` 对象)— 候选 `appmsg_base_vtbl`
  - `len 21 "TextMessageSendSource"` → `0x8279358`(== `txt_message_vtbl`)
  - `len 22 "ImageMessageSendSource"` → `0x84F96B8`(== `img_msg_vtbl`)
  - `len 22 "VideoMessageSendSource"` → `0x84F9888`(新识别 `video_msg_vtbl`)
  - **无** 专门的 `AppMessageSendSource` 分支(len 20 落入 default/error)
- ✅ 保留 `src/ForwardXMLMsg.cpp` 安全早返回(`ret=-3`)——这是护栏要求的**正确状态**,验证通过前不得移除。
- ✅ 已准备 **inert 编译护栏候选** `WeixinSend::SendAppMsg`(`src/wx_send.cpp`,由宏 `WECHAT_XML_APPMSG_RUNTIME_VERIFIED` 控制,默认不编译),复用已验证 `send_message` 路径;启用只需填入 3 个运行时验证偏移。
- ❌ **删除早返回、启用真实 XML 转发**——被卡:appmsg 内容对象(`0x718` 字节)的字段布局(XML 正文字符串偏移、接收者 wxid 偏移、appmsg 类型字段)**无法由静态分析安全推导**(RTTI 类名加密、构造函数为拷贝构造)。捏造偏移会违反护栏并崩溃微信。

### 解锁 XML 启用的唯一路径(需真实环境)

在真实微信 4.1.10.27 注入本次编译的 `version.dll`,通过 App 转发一条真实图片/链接 XML,用 IDA 运行时抓取:

1. 承载 appmsg 的**内容对象 vtable**(确认是 `0x84F9A78` 基类还是某子类型);
2. **XML 正文字符串**在内容对象中的偏移;
3. **接收者 wxid** 在内容对象中的偏移。

拿到以上 3 项运行时偏移后,填入 `SendAppMsg` 并定义 `WECHAT_XML_APPMSG_RUNTIME_VERIFIED`,即可在已验证的 `send_message` 路径上启用并删除早返回。

## 二、测试功能清单(交付后在真实微信验证)

### 自动回复

- [ ] `GET /AutoReply/config` 返回完整配置(enabled / friend_enabled / group_enabled / default_template / rules / whitelist / blacklist)。
- [ ] `POST /AutoReply/config` 写入后重启微信,`autoreply.json` 持久化生效。
- [ ] 全局 `enabled=false`:任何消息都不回复。
- [ ] `friend_enabled=false`:好友消息不回复(验证旧缺口修复)。
- [ ] `group_enabled=true` + 群消息:命中回复;`false` 时只计数不回复。
- [ ] 关键词规则命中返回对应模板;未命中用 `default_template`;`{content}/{sender}/{room}` 变量正确替换。
- [ ] 白名单/黑名单过滤生效;`/QueryDB/status` 计数(Candidates/Queued/Sent/Failed/Skipped)自洽。

### Webhook 回调

- [ ] `wx.ini` 配 `recv_url` 后,好友文本消息推送到外部服务,字段 fromid/toid/msg/msgtype 正确。
- [ ] 群消息推送含 room 与群内成员 sender。
- [ ] 断开外部服务时微信不崩溃、不卡死;`g_MessageCallbackPosts` 递增。

### 非文本接收

- [ ] 图片/语音/视频/文件/引用/系统通知分别被识别为对应 msgtype 并结构化推送。
- [ ] 群成员 wxid 与 room wxid 正确拆分。
- [ ] 连续多条混合消息后微信工作集稳定(参照约 256 MB),无崩溃/无异常增长。

### 图片发送

- [ ] `POST /SendImgMsg`(png)对方正常收到,返回 `ret=0, queued=true`。
- [ ] gif、大图、非法路径分别返回正确结果,进程稳定。

### XML 转发(当前被早返回拦截,启用后再测)

- [ ] `POST /ForwardXMLMsg` 图片 XML 对方正常收到,`ret=0`。
- [ ] 视频、GIF/emoji XML 转发成功;非法/不支持类型返回错误且不崩溃。

### DAT 解密

- [ ] `POST /Decode_Pic` 有效 `.dat` 输出可打开的图片,`ret=0`。
- [ ] 非法/空文件返回错误码,进程稳定。

### 回归

- [ ] 文本发送 `/SendTextMsg`、`/GetContact`、`/GetSelfProfile`、`/QueryDB/*` 仍正常。
- [ ] 微信重启、重新登录、退出登录后各接口状态正确,无内存异常增长。
